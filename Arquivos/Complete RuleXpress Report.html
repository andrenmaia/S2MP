Violated Quality Item(s)

A Rule Statement must include a preferred Rule Keyword
Description

A Rule Keyword can be indicated as the preferred for a Community in writing Rules. A high degree of consistency in writing Rules can be achieved by always using these select Rule Keywords.

  • Must (not)
  • May ... only (if)
Motivation

To ensure a high degree of consistency in how Statements are indicated to be Rules.
Examples

Improper example:

It is obligatory that a deposit for a reservation using a credit card be processed the same day it is received if accompanied by a signed credit card authorization form.

Proper example:

A deposit for a reservation using a credit card must be processed the same day it is received if accompanied by a signed credit card authorization form.

A Rule Statement must include a Rule Keyword
Description

A wide range of word phrases can signal that a Statement is clearly intended as a Rule. One of these Rule Keywords should appear in each Rule.

Preferred:

  • Must (not)
  • May ... only (if)

Other:

  • Obligatory, Obligation
  • Oblige, Obliges, Obliged
  • Required, Requires
  • Should (not)
  • Mandatory
  • Mandate, Mandates (as verb), Mandated
  • Need, Needs (as verb)
  • Ought (not) (to)
  • Have (to), has (to)
  • Prohibition, Prohibited, Prohibits
  • Forbidden, Forbids, Forbidding
  • (no/not/does not ... ) Permission, Permitted, Permit
  • Mustn't, Shouldn't
  • (not) Allowed, Disallowed
  • Only (if) ... may
  • Permitted .. only (if), Only (if) ... permitted, Permits ... only (if)
  • Allowed .. only (if), Only (if) ... allowed, Allows ... only (if)
  • Need not ... only (if)

  • Only (if) … not prohibited, Only (if) … not forbidden, Only (if) … not required, Only (if) … not disallowed
  • Not prohibited … only (if), Not prohibit… only (if)
  • Not forbidden … only (if), Not forbid … only (if)
  • Not required… only (if), Does not require …. only (if)
  • Not disallow… only (if), Not disallowed … only (if)

To signal explicitly that a Statement expresses some form of Rule.

Rule 13
 
A Concept must not contain rule keywords
Description

The Definition of a Concept should express its essence – what it fundamentally means to the business – not indicate or suggest any Rule or other Guidance. All Rules and other Guidance should be expressed separately from Definitions, so that change to the Rules can be managed more effectively.
Motivation

To maintain a clear separation between Definitions and Guidance Statements.
To create as stable a base of meanings as possible.
To enable rapid change.
Examples

Improper Example

  • Term: Individual Customer
  • Definition: a Customer that must not be a company, organization or legal entity

Proper Example

  • Term: Individual Customer
  • Definition: a Customer that is a Person acting in his/her own behalf
  • Rule: An Individual Customer must not be a company, organization or legal entity.

The Term defined by a Definition must not be used to initiate the Definition
Description

A Definition should never be given in the form of "An X is an X that ...", which implies there is some difference between X and that same X referenced a second time. Any given thing "X" is always simply itself. Such "self-circular definitions" cause ambiguity since they fail to express the intended meaning cleanly.
Motivation
  • To ensure that the wording of Definitions is sensible and meaningful.
  • To avoid distinctions that are logically impossible.
Examples

Improper example:

Definition of Party: a party with legal standing

Proper:

Definition of Party: a person or organization with legal standing

Concept must be defined, computed or derived
Description

Every base Term (one that is not derived or computed) must have an explicit Definition to convey its meaning. (In straightforward cases, the Definition can be based directly on a dictionary definition.) Every derived or computed Term must have a Rule indicating the logic for its derivation or computation, a Definition, or optionally, both.
Motivation

To ensure the meaning of a term has been made explicit.
To avoid misinterpretation.
Examples

Example term: Salary

Definition: Salary is ...
Computation: Salary must be computed as ... Derivation: Salary is a [other known term] that ...

A Term should be used in at least one Rule Statement or Definition
Description

A good Vocabulary is one that covers all the Concepts the business needs – and no more. If a Term has been supplanted by some other Term(s), or remains unused in building Definitions or expressing Rules, it should be removed from the Vocabulary.
Motivation

To keep a Vocabulary as trim as possible.
To ensure no Term is overlooked as the Vocabulary is developed, deployed or revised.
